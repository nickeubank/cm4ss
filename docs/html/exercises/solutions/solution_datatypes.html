    Practicing Assignment and Data Types¶

    Hello and welcome to your first R exercises!

    To complete these, please open RSTudio and create a NEW text file. Make sure that you save it with a .R at the end of the file name, since that’s how RStudio knows the file is for R code.

    Then complete the following exercises in that text file, using comments (lines that start with #) to explain what each line does.

    Exercise 1: Assigning values¶

    1. Assign the numeric value 42 to a variable called the_answer.

    [2]:

    the_answer <- 42

    1. Now create a new variable (you can call it whatever you want) that is equal to double the_answer.

    [3]:

    doubled <- the_answer * 2

    Exercise 2: Functions!¶

    1. Use print() to print out the value of the_answer

    [4]:

    print(the_answer)

    [1] 42

    1. Create a new variable called answer_divided_by_9 that is the_answer divided by 9.

    [5]:

    answer_divided_by_9 <- the_answer / 9
answer_divided_by_9
    4.66666666666667

    1. Using print() and round(), print out answer_divided_by_9 rounded to two decimal places.

    [7]:

    print(round(answer_divided_by_9, digits = 2))

    [1] 4.67

    1. Now create a variable called answer_divided_by_5 that’s the_answer divided by 5. Use round to print out answer_divided_by_5 to two decimal places. Did it work as you expected? Can you tell why things came out differently?

    [8]:

    answer_divided_by_5 <- the_answer / 5
print(round(answer_divided_by_5, digits = 2))

    [1] 8.4

    [9]:

    # It only printed one decimal because the
# second decimal is 0, and R doesn't print
# trailing zeros apparently.

    Exercise 3: Different types¶

    1. Create a variable called my_age and set it equal to your age

    [10]:

    my_age <- 35

    1. Check the class of my_age using the class() function.

    [11]:

    class(my_age)
    'numeric'

    1. Create a variable called my_dog and set it equal to the name of your dog. (If you do not have a dog, pretend you do What’s your pretend dog’s name?)

    [12]:

    my_dog <- "Trillian"

    1. What’s the class of my_dog?

    [13]:

    class(my_dog)
    'character'

    1. Now the weird one. Create a variable called seven and assign in the value "7" in quotes. What’s the class of 7? Why is it not numeric?

    [14]:

    seven <- "7"
class(seven)
    'character'

    1. What happens if you add my_age and seven?

    my_age + seven

> ERROR: Error in my_age + seven: non-numeric argument to binary operator
> Error in my_age + seven: non-numeric argument to binary operator
> Traceback:

    1. To convert seven to a numeric type, type seven = as.numeric(seven). Now check it’s class again.

    [17]:

    seven <- as.numeric(seven)
class(seven)
    'numeric'

    1. Now can you add my_age to seven?

    [18]:

    my_age + seven
    42
